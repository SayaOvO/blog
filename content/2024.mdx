又又重写了博客网站

## 关于本博客

### 为什么重写

这是我的第二个博客网站，<a href="https://sayya.vercel.app">上一个博客</a>是用 SvelteKit
写的。当时写的时候只是想熟悉下 Svelte，然而在博客初步写完直到现在，只写了一篇简单的文章之后就基本没有动过了。
关于 Svelte 我更是基本什么都记不清了，再加上用的是 js 不如 ts 写起来习惯，就萌生了重写的想法。决定开始
动手的原因是我在某个群组看到有人提及 StyleX 这个 CSS 框架，StyleX 我之前在 <a href="https://blog.skk.moe/post/refactor-my-blog-using-nextjs-app-router/">sukka 博客的这篇文章</a>
中看见过，就怀着兴趣搜索一下，发现好像才刚出来没多久，见到新东西可以玩，再加上之前有着重写这个网站的想法就动手写了

<SideNote>
  实际上，现在你看到的这个网站和写这篇文章时已经不是一个版本了
</SideNote>

顺带一提，在写到一半左右的时候，我萌生了换 [Astro](https://www.astro.build) 写的想法，但是这个想法暂时被我遏制住了 :)


### 用到的工具

这个网站虽然大部分我想要的功能已经写好的，不像上个网站一样只能算初步写好。但是仍旧毫无技术含量。用到的库有:

- Next.js

  React 最流行的框架

- ContentLayer[^1]

  内容管理工具，通过配置会将 mdx 文件转换成 JSON 数据，使用很方便，但是[暂时无维护](https://github.com/contentlayerdev/contentlayer/pull/576#issuecomment-1859741411")

- StyleX

  Meta 开源的 CSS In JS 库，应该是为了他们家的 IG 和 Facebook 等网站开发的。

- Bright[^2]

  语法高亮工具，支持 server component，这样客户端中就不会包括高亮相关的 JS 代码，减少 bundle 的大小


<SideNote>
    后来有人 fork 并维护了 ContentLayer2，我有试，但是你看到的这个最近版本还是用的 remote-mdx 这个库
</SideNote>

UI

### 博客的未来

访客统计和文章评论这两个是我考虑实现但是还没写的功能，不是多重要就等以后心血来潮的时候再搞吧。搜索就算了没有必要的。
再就是我希望本博客能变成静态的，目前暂时懒得搞。还有个 SEO 优化，完全不懂，等这个网站部署上线后就开始弄吧。剩下就慢慢填填补补，修修改改吧

然后希望自己能多用自己写的网站多写一些文章，只要写应该就会有进步的。我会为这个网站申请一个自己的域名而不是用 vercel 分配的免费域名。
最理想的状态是自己各方面能和这个网站一起进步

## 关于 2024

现在是 2024 年 1 月 2 号，这篇文章 2023 年就准备写，这篇文章跨度一年了。同时这也是个写了一年的网站 :)

我没有写年终总结和新年计划的习惯，也觉得这个比较私人不便于公开，甚至没有勇气来回顾一年的得失失失失。只是想记录下我的一些学习计划和期望

1. 准备开始试着 Remnote 和 Readwise 搭配使用，Remnote 是以间隔记忆为卖点的笔记软件，对于学过就忘的我会有帮助，实际上我一直努力想把 Anki 利用起来，
   但是全都没能坚持很久

2. 学会一门后台语言，哪一门就待定，应该会在现在比较流行的 Python, Golang, Rust 中选

3. 能够开始理解和掌握 React 的代码。在学习态度上能做到求甚解，不要偷懒

4. 多写些项目，开始参与开源项目
